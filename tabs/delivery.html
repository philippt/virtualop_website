Continous Delivery starts with efficient Development Setup.

Setting up a developer's workspace must be a one-click task.

A developer's workspace must be a close replica to the live system-

...in fact, it has to be treated from a configuration standpoint just as production.

Configuration management and handovers must be completely automated

-..no more "it works on my machine"

-..no more "just tweak this parameter and it works"

Continuos delivery is enforced by enabling and encouraging continous commits, integration and testing.

COMMIT, TEST, INTEGRATE : just do it, several times a day.

You might not need to roll out features every few hours, but if needed, it must be possible.

It must be possible to set up your whole system from scratch as a one- click from dev's latest commit.

Anytime.
