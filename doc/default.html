The virtualop is a framework for automating the setup, deployment and operations of web applications.


It is written using the Ruby programming language, consists of a relatively small core, a command line interface, a web interface and a growing number of plugins and services.

The plugins contain logic for setting up the virtualop platform, which is a set of virtual machines deployed across 3 different physical hosts (two for production, one for staging). This platform is self-deploying, i.e. it automatically deploys new versions of itself onto the staging environment, tests them there and updates itself on successful completion of all tests. A similar mechanism can be used to setup machines for development and integrate changes prepared there.

The framework's design makes it easy to integrate a variety of services into the platform, and if any part of the platform does not fit your needs, you can replace it with your own version.

The virtualop is still in an alpha stage - parts of the functionality are completed, including:

  • installation: setup new virtual machines with CentOS or Ubuntu
  • deployment: install your projects from github, through git, subversion or from ownCloud
    • github: if you connect your github account, the virtualop initializes new repositories and manages SSH keys for you.
    • ownCloud: The minimal platform comes with a built-in owncloud server, and the virtualop prepares development machines with an owncloud client connected to it.
  • canned services deploy #{services_count} services with one click
  • monitoring: Nagios configuration is automatically generated for new machines

What is missing for public consumption is:

  • documentation
  • security
